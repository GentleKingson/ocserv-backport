## 1. 背景与问题

### 1.1 现象

`docs/trixie-builder-dryrun-runbook.md` 存在**两套并行且错位 1 的编号系统**：

- **顶层标题**用「第 N 步」：第 1 步 / 第 2 步 / 第 3 步 / 第 4 步
- **子节标题**用残留的旧编号 X.Y，且 X 比对应步号**大 1**：

| 顶层步号 | 子节实际编号 | 期望（与步号对齐） | 是否错位 |
|---------|-------------|------------------|---------|
| 第 1 步 | 1.1 / 1.2 / 1.3 | 1.x | ✅ 一致 |
| 第 2 步 | **3.0** / 3.1 / 3.2 / 3.3 / 3.4 | 2.x | ❌ 错位 |
| 第 3 步 | **4.0** / 4.1 … 4.6 | 3.x | ❌ 错位 |
| 第 4 步 | **5.1** / 5.2 … 5.5 | 4.x | ❌ 错位 |

### 1.2 根因

文档早期用「引言=0、裸机=2、配置=3、运行=4、dry-run=5」的连续编号。后来顶层标题改为「第 1~4 步」（引言不再编号、裸机从 1 起），但**第 2~4 步的子节 X.Y 没跟着改**，残留了旧编号。

### 1.3 对读者的影响

读者在同一句话里同时看到两个打架的数字，例如：

- 正文 L31：`sbuild group 刷新那个特定节点（第 3 步 4.3）` —— "3" 和 "4.3" 到底哪个才对？
- L671：`回到：第 3 步 4.3（重新登录 / newgrp sbuild）` —— 同样冲突

更糟的是，**部分交叉引用指向根本不存在的小节**（来自更早一版，当时裸机准备是 2.x）：

- L284：`回到第 1 步 2.1 扩容` —— 第 1 步没有 2.1（只有 1.1/1.2/1.3）
- 附录 B L794–798：`第 1 步 2.3`、`第 1 步 2.4` 等全部基于已失效的旧编号

### 1.4 范围边界

**本设计只改文档编号与交叉引用，不改：**

- 文档的章节顺序、内容、命令、占位符语义
- `make dry-run` 流水线表里的「步骤 1~8」（L631–640）—— 这是流水线内部步骤，**不是文档章节**，保留不动
- 5.3 失败定位表里的「步骤 1/4/6/7」—— 同样指流水线步骤，保留
- 任何脚本/代码文件

## 2. 目标

1. **子节编号与所在步号对齐**：第 N 步的子节统一为 N.x
2. **保留 .0 作概述节**：第 2/3 步的现有 .0（"两个 dry-run 区分"、"阶段执行规则"）作为章下概述节保留，后续从 .1 起
3. **修复所有失效/错位交叉引用**：包括正文引用、附录 B 占位符表、附录 C 引用
4. **流水线步骤编号不动**：明确区分"文档章节编号"与"流水线步骤编号"两个概念

**非目标：** 重写文档内容、调整章节顺序、新增/删除章节。

### 3.3 明确不动的内容

以下编号**故意保留**，因为它们指代流水线内部步骤，与文档章节编号是两套体系：

为避免读者混淆这两套编号，**不在本次改动里加额外说明**（避免改动范围蔓延）；流水线表自身的"步骤"列已经够清楚。如果未来发现读者仍混淆，再单独立项加一句提示。

## 4. 验证

### 4.1 完成判定（标题检查 + 引用检查两类）

**不做过宽的"全文不存在 3.1/4.1"式 grep** —— 版本号（`1.5.0-1`）、命令、路径等上下文会造成误报。改用以下两类定向检查：

**A. 标题检查（grep `^### ` 行）：**

1. 确认不再存在旧的 `### 3.x / ### 4.x / ### 5.x` 标题
2. 确认第 2 步下只出现 `2.x`、第 3 步下只出现 `3.x`、第 4 步下只出现 `4.x`
3. 每步子节编号在步内单调递增

**B. 引用检查（以 spec §3.2 逐条清单为基准）：**

1. 逐项核对 §3.2 表 B/C 的每一条引用都已按新编号更新
2. **重点复核高风险区域：**
   - 失败定位区（5.3→4.3，L665–690）的"第 3 步 3.3 / 3.4 / 3.5"
   - 附录 B 七格"出现位置"列（L793–799）
   - 阶段间动作节（4.3→3.3）被引用处（L31、L346、L405、L407）
3. 附录 B 每一格的"出现位置"都能在正文找到对应编号的小节

**C. 行数检查（辅助）：** 本次只改编号不改内容，行数应在 ±5 行内（标题行只改数字字符）。

- 文档 git diff 应**只动编号和交叉引用文本**，不出现内容段落的增删
